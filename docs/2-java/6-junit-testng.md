## JUnit/TestNG 是框架吗？

这个问题的回答是是也不是，原因如下:

- 是的原因: 确实起到了管理测试的功能，给测试代码提供了一套解决方案
- 不是的原因: 两个都过于抽象，对于实际的业务起不到任何帮助

## 手动写一个最简单的类似的测试框架

- 注解@Test
- 运行:Runner
- 记录结果
- 输入结果

整个例子可以看到，这些管理的只是代码层面的用例，和实际的业务场景完全没有关系
实际的业务场景工作量会比写一个框架的工作量和维护量都要大.
